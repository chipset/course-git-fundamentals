# Git Fundamentals

This is a generic Instrktr course for teaching practical Git.

The course is designed to teach both the commands and the reasons behind them. Each step asks learners to notice Git's model: the working tree, staging area, commit history, branches, remotes, and recovery tools. The goal is not command memorization; it is knowing which part of Git's model you are changing and why.

It covers:

- repository setup
- staging and commits
- diff and log inspection
- `.gitignore`
- branching and merging
- merge conflict resolution
- local remotes, fetch, pull, and push
- file restore/reset
- revert versus reset
- stash
- reflog recovery

The course uses local repositories only. Remote lessons create a bare repository in a sibling directory so learners can practice `origin`, `push`, `fetch`, and `pull` without GitHub credentials.

For deeper reference, use the official [Pro Git book](https://git-scm.com/book/en/v2), the [Git command reference](https://git-scm.com/docs), and the printable [Git cheat sheet](https://git-scm.com/cheat-sheet.pdf).
